# PRODUCTIZATION REFACTOR V0.4.5

## 目标

这一轮不是重新做一次“初级友好化”，而是基于 `0.4.4` 已有骨架，继续把系统往 **真正可用的产品入口** 推进。

坚持的约束：

- 不推翻 runtime / state / orchestration 核心
- 不另起炉灶做新框架
- 不只改 README / help / prompt
- 不只做一层静态 UI 外壳
- 继续以普通用户第一次使用成功为中心

---

## 对 0.4.4 的判断

### 已经解决的部分

- 有了更友好的 `quickstart / init / demo / doctor / status / ui`
- 有了人话版 CLI 输出
- 有了基础 Web UI
- 有了 onboarding 与 `START_HERE.md`
- 有了术语翻译和下一步推荐

### 还没彻底解决的部分

- UI 还更像“状态面板”，不是“项目主入口”
- 普通用户仍然要在多个地方猜“接下来做什么”
- 很多列表只能看，缺少详情页和操作入口
- 任务、run、artifact、session 的可读性和可恢复性不够
- “等待确认 / backlog / dispatchable” 还会被用户误解为故障
- 首页还没有承担“第一次进入控制台”的角色

---

## 本轮重构策略

### 1. 以现有骨架继续做深，不推翻

保留：

- CLI
- runtime / planner / scheduler / evaluator / sqlite 同步能力
- 当前 workspace 布局
- 当前命令体系

增强：

- `research_os/webapp.py`
- `research_os/ux.py`
- `research_os/bootstrap.py`
- 文档与版本说明

### 2. 优先处理普通用户主路径，而不是工程内部效率

先解决：

- 首页主入口
- 项目页总览
- 详情页
- 下一步推荐
- 健康检查解释层
- 空状态 / 错误恢复

### 3. 尽量低复杂度换高价值

没有新增新的 state 体系，也没有新增复杂前后端栈。
在现有 `http.server` 风格 Web UI 上继续增强，换取更高产品化程度。

---

## 实际改动

### UX 层
- 新增更完整的项目进度、阶段路径、阶段就绪度、最近里程碑、笔记预览
- 新增任务 / run / artifact / session / note 详情数据聚合函数
- 新增体验状态判断：刚开始 / 正在推进 / 等待确认 / 需要修复
- 把流程性提醒从故障型告警里拆分出来

### Web UI
- 首页升级为项目控制台
- 增加项目级 tab：总览 / 待办 / 任务记录 / 输出成果 / 最近进展 / 健康检查
- 增加详情页：任务、run、输出文件、项目材料、推进记录
- 增加高频动作按钮：推进、批准、重试、取消、审计、doctor、本地执行器
- 增加新手模式 / 高级模式
- 增加空状态、引导卡片、最近项目、环境状态

### Onboarding
- `START_HERE.md` 更新为 UI-first 流程
- 明确“等待确认通常不是故障”
- 把第一次路径收敛成“打开 UI → 推进一步 → 看下一步 / 最近进展”

### 文档
- README / QUICKSTART / CHANGELOG 升级到 `0.4.5`
- 新增本文件，记录这一轮的产品化深化内容

---

## Breaking changes

这一轮没有推翻已有核心结构，也没有移除旧 CLI。

潜在行为变化：

- Web UI 现在会把“等待确认 / backlog / 可调度任务”视为中性流程状态，而不是默认故障
- 首页和项目页默认更偏向普通用户解释层，原始信息更多收进高级模式
- `START_HERE.md` 的引导顺序改为优先 UI，而不是优先 CLI 状态查看

---

## Migration

### 对已有项目
通常不需要迁移数据。

### 对使用 CLI 的高级用户
原命令继续可用；只是推荐把普通用户引导切换到 `ros ui`。

### 对依赖旧 Web UI 截图或人工操作 SOP 的团队
建议更新内部说明，改为：

1. 先从首页进入项目
2. 在总览页执行高频动作
3. 需要底层信息时再切换到高级模式

---

## 下一步建议

如果继续做 `0.4.6+`，建议优先：

1. 更强的 run 结果对比与差异解释
2. 更完整的“恢复路径”按钮化
3. 项目目标与输入表单可视化编辑
4. 审批原因与风险解释卡片化
5. 更强的最近成果卡片与项目摘要封面
