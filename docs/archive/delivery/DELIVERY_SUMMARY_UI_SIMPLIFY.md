# UI 简化优化说明

本次直接对项目的主界面做了一轮“再减法”优化，目标不是加新风格，而是把已有界面继续压到更专注、更轻、更少干扰。

## 这次主要做了什么

### 1. 项目首页再压一层
- 去掉右侧两张说明卡，避免首页视觉分叉。
- 保留一个明确主入口：**继续当前工作**。
- 把“当前步骤 / 项目进度 / 最近采纳”压成一行轻量摘要。
- 把“当前最该推进”改成一张主聚焦卡，减少解释块数量。
- 路线、最近采纳、最近更新都退到下方折叠区，主视线更稳定。

### 2. 工作页结构继续单线化
- 头部改成：**当前一步 + 路线切换 + 抽屉按钮**，不再堆太多说明。
- 中央编辑区只保留三层：
  - 当前目标 / 本步交付
  - 主输入
  - 当前产物
- 输出侧栏从多张分散卡片合并成一张摘要卡，减少“右侧信息墙”感。
- 保留历史、对比、交接、活动流，但继续放在折叠区。
- 抽屉仍保留，但视觉压平，优先让中间工作区成为主角。

### 3. 总控页减重
- “运行方式”改成折叠说明，而不是常驻大块卡片。
- 保留总览、活动、主输出、模板、AI 接入、文件库、交接，但统一成更轻的表面风格。

### 4. 视觉风格整体压扁
- 背景、面板、边框改成更中性、更低对比度的浅色体系。
- 阴影明显减弱，减少“很多浮层同时存在”的复杂感。
- 卡片圆角、按钮、badge、tab 一起收敛，统一成更克制的语言。
- 重点从“装饰提示”转向“信息顺序”。
