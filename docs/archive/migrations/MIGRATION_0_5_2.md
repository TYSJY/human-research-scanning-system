# Migration 0.5.2

## 核心变化

0.5.2 不是继续扩“大而全控制台”，而是在 0.5.1.1 的基础上把 UI 收敛成更接近日常论文生产的浅色工作台：

- 全局主题改为浅色
- A/B/C 中间工作区成为绝对主区域
- 左栏步骤树和右栏文件区降噪
- 低频操作下沉到折叠区
- D 线增加锚点导航，Provider / 模板 / 交接管理改成折叠管理区

## 使用上最明显的变化

### 1. 首页和侧栏更轻
- 已有项目保留在侧栏
- 新建项目表单改成折叠区
- 首页不再展示偏“控制台式”的深色大 Hero

### 2. A/B/C 步骤页更聚焦
- 当前步骤区增加摘要统计卡：进度 / 尝试数 / 文件引用 / 下一步
- Prompt 和运行动作仍在主区，但把模板保存、执行设置、人工备注下沉到折叠区
- 历史尝试版本和子步骤新增入口改为折叠区

### 3. 右侧文件区更安静
- 上传 / 引用 / 导入外部结果都改成折叠块
- 默认先看当前文件列表，不让表单堆满整列

### 4. D 线更像控制中心
- 顶部增加总览 / 主版本 / 文件库 / Provider / 模板 / 交接锚点导航
- Provider / 模板 / 交接统一改成折叠管理区

## Windows 脚本变化

`install_windows.bat` / `start_windows.bat` / `start_windows.ps1` 现在会更明确地识别 WindowsApps 的假 Python 并给出提示，不再静默失败。

## 对旧项目数据的影响

- 不影响已有项目数据结构
- 不需要迁移项目目录
- 共享文件库、Step / Attempt / Asset / Package / Handoff 模型保持兼容
