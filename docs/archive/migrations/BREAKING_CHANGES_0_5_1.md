# Breaking Changes in 0.5.1

0.5.1 没有引入大范围结构性 breaking changes，但有几处行为变化需要注意。

## 1. Prompt 模板现在支持变量渲染

模板里如果写了：

- `{{project_brief}}`
- `{{step_title}}`
- `{{step_goal}}`
- `{{input_assets}}`

那么在“套用模板”时会直接被渲染成当前步骤上下文。

这意味着：

- 旧模板不会坏
- 但新模板可以写成“变量化模板”，套用后会自动展开

## 2. “设为主文件”不再只是文件动作

当你把某次尝试的输出固定为主文件时，系统现在会同时：

- 把这次尝试设为当前最佳版本
- 把这次尝试标记为 `preferred`

这是有意的行为收敛，因为对普通用户来说：

“把某次尝试的文件固定为主版本”
通常就意味着
“这次尝试本身是当前优选”。

## 3. 状态写入改成原子写入

`save_json()` / `write_text()` 现在会先写临时文件，再替换目标文件。

这不是 API breaking change，但会改变文件写入方式，有助于降低异常中断时状态文件损坏的概率。

## 4. D线文件库管理页不再只是静态列表

现在 D线文件库页默认带：

- 搜索
- 分区筛选
- 模块筛选
- 类型筛选
- 主版本 / 待整理筛选
- 预览

这属于 UI 行为变化，不影响底层数据。

## 5. 尝试版本对象上增加人工审阅字段

如果你有依赖旧 `Attempt` 结构的外部脚本，请注意新增字段：

- `review_decision`
- `review_score`
- `review_tags`

旧脚本通常不会因此报错，但如果你做了严格 schema 校验，需要同步放宽。
